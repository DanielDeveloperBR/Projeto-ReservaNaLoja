Visão Geral

Frontend estático para reservas em loja. Estruturado em HTML/CSS/JS sem dependências de build. Código focado em responsividade, menus reutilizáveis e formulários (login/cadastro/agendamento).

Validação no cliente; se houver integrações, manter endpoints e métodos documentados aqui.

Padrões de Código

  • HTML semântico; preferir classes a ids como ganchos de JS.
  • CSS por componentes; media queries no fim do arquivo.
  • JS modular; uma função por responsabilidade; evitar globais.
  • Nomenclatura: kebab-case (CSS) e camelCase (JS).

Como Executar

# Local
# 1) Abra index.html no navegador
# ou
# 2) VSCode → Extensão "Live Server" → Open with Live Server

Em produção, garanta caminhos relativos para assets (./css, ./js, ./img).
